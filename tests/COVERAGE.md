# Test Coverage Policy & Waiver Registry

Coverage is measured by `pnpm test:coverage` (Vitest 4 + v8 provider).
Reports land in `coverage/` (git-ignored). See
`tests/COVERAGE-BASELINE.md` for the current enforced floors.

**Enforcement stance (post-2026-04-22 campaign):** `pnpm test:coverage`
fails locally and in CI when any of the four thresholds regresses below
floor (lines 75 / branches 67 / functions 75 / statements 73). Floors
sit 2 pp under achieved to absorb noise; raise them as headroom grows.
Waivers below remove paths from the denominator entirely.

## Waiver Policy

A path is **waived** if:

1. **Auto-generated** — regenerated by tooling, tests would pin the
   generator output, not behavior. (Cite generator + regen command.)
2. **Third-party shim** — thin glue around an upstream dep whose
   own tests cover the logic.
3. **Platform code** — untestable without the target environment
   (Tauri runtime, WebKit window, OS APIs).
4. **Entry points / bootstrap** — top-level wiring with no branching
   logic worth testing in isolation (e.g., `main.ts`).

Anything else must be tested (or waived with an explicit entry here).

## How to Add a Waiver

1. Open a PR that edits both:
   - `vitest.config.ts` → `coverage.exclude` (add the glob)
   - this file → add a row to the table below
2. The PR description MUST cite:
   - Which of the 4 waiver categories applies
   - Why tests are infeasible (not just inconvenient)
   - Expiry date or the word `permanent`
3. Reviewer checks that the rationale actually fits the category.
   "We didn't get around to writing tests" is not a rationale.
